# STEP 03. 자료 업로드 v4 가이드

## 이번 버전에서 되는 것

- 자료 업로드 화면에서 CSV, XLSX, XLS 파일 선택
- 업로드 유형 선택: 은행 입출금 / 카드 사용내역 / 파로스 분개 / 자산·부채 증감
- 브라우저에서 파일 미리보기 생성
- 거래일자, 거래처/적요, 금액, 현금흐름, 사업부 1차 자동 인식
- `검증대기 저장` 시 Supabase에 저장
  - `upload_batches`: 업로드 이력
  - `upload_raw_rows`: 원본 행 및 자동 인식 결과
  - `transactions`: 은행/카드/파로스 거래 1차 생성
- 최근 업로드 이력 표시

## 먼저 실행해야 하는 SQL

Supabase SQL Editor에서 아래 파일을 실행한다.

```text
 database/migrations/20260617_uploads_v4.sql
```

성공하면 `Success. No rows returned`가 나온다.

## 권한

- `admin`: 업로드 가능
- `finance`: 업로드 가능
- `executive`: 조회 중심, 업로드 제한
- `viewer`: 조회 중심, 업로드 제한

## 운영 기준

- 이번 v4는 업로드 MVP다.
- 업로드된 거래는 모두 `확인필요` 상태로 저장한다.
- 대시보드 최종 반영/확정/수정 이력은 다음 단계에서 연결한다.
- 1회 업로드는 최대 5,000행으로 제한한다.

## 다음 단계

2. 확인필요 항목 수정 화면

4. 확정 처리 후 대시보드 반영
5. 파일 원본 자체를 Supabase Storage에 보관
