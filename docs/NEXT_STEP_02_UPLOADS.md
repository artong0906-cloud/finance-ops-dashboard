# 2단계 예고: 업로드 기능 구현

2단계의 목표는 은행/카드 로우데이터를 실제로 업로드하고 검증대기 목록까지 만드는 것입니다.

## 구현 순서

1. Supabase Storage bucket 생성
2. upload_batches insert 기능 생성
3. CSV/XLSX 파일 업로드 UI 구현
4. 업로드 파일 미리보기 구현
5. 컬럼 자동 인식
6. 은행 계좌번호 기준 사업부 자동 부여
7. 카드 예산그룹/공통사용분 기준 임시 분류
8. 검증 이슈 표시
9. 확인 후 transactions에 반영

## 검증 항목

### 은행

- 계좌번호가 bank_account_master에 있는가?
- 입금/출금 구분이 정상인가?
- 내부이체 후보가 있는가?
- 카드대금 결제가 비용으로 중복 반영되지 않았는가?

### 카드

- 예산그룹이 잡혔는가?
- 공통사용분 여부가 잡혔는가?
- 분개 전/분개 완료 상태가 구분되는가?
- 대표자 카드가 별도 표시되는가?
